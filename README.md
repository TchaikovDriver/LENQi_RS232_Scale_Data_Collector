# RS232 电子秤数据采集

通过 USB-RS232 连接电子秤，采集称重数据并自动生成时间-重量曲线图。

## 环境要求

- Python 3.10+
- USB-RS232 转接线
- 支持 RS232C ASCII 协议的电子秤

## 安装依赖

```bash
pip install -r requirements.txt
```

## 运行程序

```bash
python main.py
```

## 使用步骤

### 1. 连接串口

1. 将电子秤通过 USB-RS232 接入电脑
2. 运行程序后，会自动扫描并列出可用串口
3. 输入对应数字选择端口
4. 连接失败时会提示错误，可重新选择；连接成功后进入主界面

示例：

```
可用串口：
  1. /dev/cu.usbserial-1410
请输入数字选择端口: 1
已成功连接 /dev/cu.usbserial-1410
```

### 2. 开始采集

连接成功后，输入 `1` 开始一轮采集：

```
输入 1 开始采集，输入 q 退出: 1
```

### 3. 采集流程

1. 程序等待电子秤称重，当重量 **> 0.1** 时自动开始记录
2. 每 **0.1 秒** 采样一次重量和时间
3. 重量发生变化后继续记录；若重量连续 **5 秒** 不再变化，则结束本轮采集
4. 程序自动生成点线图并保存到 `results/` 目录
5. 图表生成完成后，可再次输入 `1` 开始新一轮采集（各轮数据互不影响）

### 4. 退出程序

输入 `q` 退出，程序会自动关闭串口连接。

## 输出结果

- 保存目录：`results/`
- 文件命名：`年月日时分.png`，例如 `202607141104.png`
- 图表内容：
  - 横轴：时间（毫秒，从采集开始时刻起算）
  - 纵轴：重量（克）
  - 标题：Flow Curve

## 通信协议

| 项目 | 说明 |
|------|------|
| 接口 | RS232C |
| 波特率 | 9600 |
| 数据位 | 8 |
| 校验位 | 无 |
| 停止位 | 1 |
| 数据格式 | 8 字节 ASCII + `0x0A`（换行） |

数据示例：

- 正数 `"    1.23\n"` → `1.23`
- 负数 `"   -1.23\n"` → `-1.23`

去掉前后空格后，解析为带小数点的数值。

## 常见问题

**未检测到串口**

- 检查 USB-RS232 线是否插好
- 确认驱动已安装
- 按回车重新扫描端口列表

**读取重量失败**

- 确认选择了正确的串口
- 确认电子秤已开机且通信参数为 9600 / 8N1
- 检查数据线连接是否稳定

**采集一直不结束**

- 重量需先发生变化，之后连续 5 秒保持不变才会结束
- 若重量持续波动，采集会一直进行直到稳定

## Windows 可执行文件

使用 PyInstaller 打包：

```bash
pip install pyinstaller
pyinstaller --onefile --console main.py
```

生成的文件位于 `dist/main.exe`。

**注意：必须使用 `--console` 模式，不要使用 `--windowed`。**

本程序依赖 `input()` 进行交互，`--windowed` 会隐藏控制台并导致 `lost sys.stdin` 错误。

建议在 **cmd** 或 **PowerShell** 中运行：

```powershell
.\main.exe
```

不要直接双击运行，否则窗口可能在出错后立即关闭，不便查看日志。
